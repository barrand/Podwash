# Slice 02 matching-engine fixtures — provenance

> Named `PROVENANCE.md` (not `README.md`) on purpose: the Xcode
> `PBXFileSystemSynchronizedRootGroup` flattens bundled resources into the test
> bundle root, so a second `README.md` here would collide with
> `../README.md` at build time. Content is the fixtures README for this folder.

**Hard QA rule:** these goldens have provenance **independent of the code under
test**. They were **hand-computed from `docs/specs/matching-spec.md` §5–§8 on
2026-07-08**, transcribed by hand into JSON. They were **NOT** generated by
`WordMatcher` / `IntervalBuilder` (the code they verify). Regenerating a golden
from the implementation it tests would make the assertion circular and is
forbidden.

Target set for both cases (example-local): `{ "shit", "damn" }`.

## `spec-section8.*` — the §8 worked example

- `spec-section8.input.json` — the 5-word transcript from spec §8 (`TimedWord`
  schema per ADR-000 §4).
- `spec-section8.expected.json` — hand-computed merged intervals.

Hand computation (spec §3–§6):

| Token | Normalized | Match? |
|-------|-----------|--------|
| `Well,` | `well` | no |
| `shit,` | `shit` | yes |
| `damn` | `damn` | yes |
| `shipment` | `shipment` | no (exact membership — no substring match on `ship`/`shit`) |
| `Shit!` | `shit` | yes |

Pad matches (start − 0.080, end + 0.120; all durations ≥ 0.180, no expansion):

- `shit,` [1.00, 1.30] → [0.92, 1.42]
- `damn`  [1.50, 1.75] → [1.42, 1.87]
- `Shit!` [3.00, 3.20] → [2.92, 3.32]

Sort (already sorted) + merge (`start <= previous.end` merges; touch counts):

- `[0.92, 1.42]` opens.
- `[1.42, 1.87]`: 1.42 <= 1.42 → merge → `[0.92, max(1.42, 1.87)] = [0.92, 1.87]`.
- `[2.92, 3.32]`: 2.92 > 1.87 → new interval.

Golden: `[{0.92, 1.87}, {2.92, 3.32}]`.

## `clamp-expansion.*` — t=0 clamp + midpoint-expansion quirk

- `clamp-expansion.input.json` — one matched word `shit` at `[0.00, 0.05]`.
- `clamp-expansion.expected.json` — hand-computed final interval.

Hand computation (spec §5 quirk):

- padded = `[max(0, 0.00 − 0.080), 0.05 + 0.120]` = `[0.00, 0.17]`.
- duration 0.17 < 0.180 → expand: midpoint = 0.085, half = 0.090.
- new start = `max(0, 0.085 − 0.090)` = `max(0, −0.005)` = **0.000**.
- new end = `0.085 + 0.090` = **0.175** (NOT re-extended to compensate for the
  start clamp — this is the documented quirk).

Golden: `[{0.000, 0.175}]` — still 0.175 s (< MIN_CENSOR_SECONDS).

Comparison tolerance for all Double assertions: **±0.0005 s**.
